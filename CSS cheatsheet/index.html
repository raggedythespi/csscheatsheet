        By default, position is set to static. It positions based on the layout in the flow. 
    
        Position absolute vs Position relative
        Position relative comes in without changing the layout around it. It leaves a gap for any box/text where it would have been had it not been position.
        While position absolute does not leave any gap for an element where it had been. The position is totally based on its parent position.

Position fixed

An element with psoition fixed is positioned relative to the viewport which means it always stays in the same place even if the page is scrolled. A fix element does not leave a gap in the page where it would normally have been located.

This div element has position: fixed;

Position sticky!

An element with position sticky is positioned based on the user's scroll position. Such an element toggles between relative and fixed, dependig on the scroll position. It is positioned relative until a given offset position is met in the viewport - then it sticks in place like position fixed. Internet explorer does not support sticky positioning and safari requires a -webkit- prefix.

CSS Layout - the Z index property

When elements are positioned, they can overlap the other elements. The z-index property specifies the stack order of an element. (which element should be placed infront of or behind the othes). An element can have a positive or negative stack order.

Because the z-index property has a negative value, the image would be behind the text.

Z index element works only on positioned elements or flex items (elements that are direct children of display flex elements).

An element with a greater stack order is always above an element with lower stack order. See an example below!!

Black box (z-index: 1)
Gray box (z-index: 3)
Green box (z-index: 2)

Note:If two positioned elements overlap each other without a z-index specified, the elements defined in the last in the HTML code will be shown on top. Such as green box in an example above will remain on top if the z-index is unspecified.

The overflow property specifies whether to clip the content or to add scrollbars when the content of an element is too big to fit in the specified area.

        The overflow property has the specific values:
        (a) Visible - This value is default and the overflow is not clipped. The content renders outside the element's box.
        (b) Hidden - The overflow is clipped, and the rest of the content will be invisible
        (c) Scroll - The overflow is clipped, and a scrollbar is added to see the rest of the content.
        (d) Auto - Similar to scroll, but it adds scrollbar only when necessary.

CSS Layout - float and clear

The CSS float property specifies how an element should float while the clear property specifies what element can float beside the cleared element and on which side.

The float property

The float property is used for positioning and fromatting content. For example, let an image float left to the text in a container. The float property can have one of the following values: left, right, none, inherit. The float property can be used to wrap text around images.

Display inline-block explained!

Comapred to the inline element, the major difference between the inline block element and inline element is that the inline-block element allows you to set the width and height of an element. Plus with the inline-block elements, the top and bottom margin/paddings are respected but with display: inline; they are not.

Compared to display: block; the major difference is that the inline-block does not add a line break after the element, so the elements can sit next to other elements
